# Premium Request Usage Guide

This guide covers understanding and managing GitHub Copilot's premium request system, including model selection strategies and cost optimization.

## Understanding Premium Requests

GitHub Copilot offers different AI models with varying capabilities and costs. Some models are included with unlimited usage, while others consume "premium requests" from a monthly allowance.

### Model Categories

#### Unlimited Models (0x Multiplier)
- No request limits
- Suitable for everyday coding tasks
- Fast response times

#### Premium Models
- Count against monthly allowance
- More powerful capabilities
- Better for complex tasks

### Checking Your Usage

1. Navigate to [github.com/settings/copilot](https://github.com/settings/copilot)
2. View your current usage statistics
3. Check remaining premium requests

## Model Selection Strategies

### When to Use Unlimited Models

- **Quick code completions** - Simple suggestions and autocomplete
- **Basic explanations** - Understanding straightforward code
- **Routine refactoring** - Small, focused changes
- **Learning exercises** - Exploring and experimenting

### When to Use Premium Models

- **Complex problem-solving** - Architectural decisions, algorithm design
- **Deep code analysis** - Security reviews, performance optimization
- **Multi-file features** - Large-scale code generation
- **Advanced reasoning** - Complex business logic, edge cases

### Auto Model Selection

The **Auto** option provides:
- Automatic model selection based on task
- 10% discount on premium requests
- Optimal balance of capability and cost
- Simplified workflow

**Recommended for**: Users who want cost optimization without manual model selection.

## Cost Optimization Tips

### 1. Start with Simpler Models
Begin with unlimited models and upgrade only when needed.

### 2. Provide Better Context
Better prompts reduce the need for follow-up queries:
- Open relevant files
- Be specific in requests
- Reference existing patterns

### 3. Use Ask Mode First
Explore and plan in Ask mode (free) before executing changes.

### 4. Batch Related Tasks
Group similar requests to minimize model switches.

### 5. Monitor Usage Patterns
Track which tasks consume the most requests and optimize.

## Model Comparison by Task Type

| Task Type | Recommended Model | Why |
|-----------|------------------|-----|
| Code completion | Unlimited | Speed and frequency |
| Simple explanation | Unlimited | Low complexity |
| Complex refactoring | Premium/Auto | Better reasoning |
| Security analysis | Premium | Deeper analysis |
| Architecture design | Premium | Strategic thinking |
| Test generation | Auto | Balance of cost/quality |
| Documentation | Unlimited/Auto | Generally straightforward |

## Monitoring and Alerts

### Setting Up Usage Alerts

1. Check if your organization has usage monitoring
2. Set personal reminders mid-month
3. Review usage patterns regularly

### What to Do If Running Low

1. Switch to unlimited models for routine tasks
2. Save premium models for critical work
3. Optimize prompts to reduce follow-ups
4. Wait for monthly reset if possible

## Team Considerations

### Organization Policies

- Check organization limits and policies
- Understand shared vs. individual allowances
- Follow team guidelines for model usage

### Best Practices for Teams

1. Establish guidelines for premium model usage
2. Share effective prompts that reduce requests
3. Document which tasks warrant premium models
4. Review team usage patterns monthly
